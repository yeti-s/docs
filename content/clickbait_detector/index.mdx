한양대학교 자연어처리 강의 팀 프로젝트로 낚시성 기사에 대해 ~~내용을 요약해주는~~ 제목을 재생성하는 서비스를 진행하기로 하였어요.  
`기존 낚시성 기사에 대한 제목 재생성으로 목표를 변경하여 일반 기사와 같은 형태로 제목을 보고 유저가 글을 읽을지 말지 판단하는 형태로 변경했습니다.`

# 기획

프로젝트를 진행하기 위해서 크게 세 가지로 작업을 나눌 수 있어요.
* 낚시성 기사 탐지
* 기사 내용을 통한 제목 생성
* 시각적으로 모델을 활용할 수 있는 Interface

## 낚시성 기사 탐지

기사의 제목과 내용을 이용하여 낚시성 여부를 판단하는 모델을 구현 할거에요.

기존 BERT 모델을 이용하여 낚시성 여부를 판단하려 하였으나,
기사 제목 + 내용이 한꺼번에 토큰화되어 모델의 입력으로 들어가기 때문에 BERT의 최대 토큰 수 제한이 정확도를 하락시킬 수 있다고 판단했습니다.
따라서 LongFormer의 self-attention 메카니즘을 BERT에 적용하고 최대 토큰 수를 늘리는 방식을 활용하여 구현을 진행도록 기획하였어요.  

[build a "long" version of pretrained models](https://github.com/allenai/longformer/blob/master/scripts/convert_model_to_long.ipynb)  
[낚시성 기사 데이터 (AI-Hub)](https://www.aihub.or.kr/aihubdata/data/view.do?currMenu=&topMenu=&aihubDataSe=data&dataSetSn=71338)

## 제목 생성

기사의 내용을 통해 제목을 생성하는 모델을 구현합니다.

초기에는 생성형 모델(GPT2)을 통한 학습으로 구현하려 하였는데, BART 모델이 텍스트 이해와 생성 작업에 좋은 성적을 낸다는 것을 보았어요.
따라서 이를 활용하여 구현을 진행하도록 기획했어요.
